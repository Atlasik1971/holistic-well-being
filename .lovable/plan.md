

## Удаление Telegram-бота для уведомлений

Полностью отключаю систему уведомлений в Telegram о новых заявках, сообщениях и вопросах из чата. Заявки продолжат сохраняться в базе и будут доступны в админке — пропадут только пуш-уведомления в Telegram.

### Что будет удалено

1. **Триггеры в базе данных** на таблицах `bookings`, `contact_messages`, `chat_leads` — они вызывали отправку в Telegram при каждой новой записи.
2. **Функция БД** `notify_telegram_on_insert()` — больше не нужна.
3. **Edge-функция** `notify-telegram` — серверный код, отправлявший сообщения в бота.
4. **Подключение коннектора Telegram** к проекту — секрет `TELEGRAM_API_KEY` будет убран из проекта (само подключение к боту в воркспейсе сохранится, чтобы можно было использовать в других проектах при желании).
5. **Секрет** `ADMIN_TELEGRAM_CHAT_ID` — больше не используется.

### Что останется без изменений

- Формы на сайте (бронирование, контакты, чат-виджет) — продолжают работать.
- Записи продолжают сохраняться в БД и видны в админке (`/admin/bookings`, `/admin/messages`, `/admin/chat-leads`).
- Сам бот **@Zayavka_sait_coder_bot** в Telegram остаётся живым — его никто не удалит, просто сайт перестанет ему писать. Если захотите удалить и сам аккаунт бота — это делается вручную через @BotFather командой `/deletebot`.

### Технические шаги (для справки)

```text
1. SQL миграция:
   DROP TRIGGER trg_notify_telegram_bookings ON bookings;
   DROP TRIGGER trg_notify_telegram_contact_messages ON contact_messages;
   DROP TRIGGER trg_notify_telegram_chat_leads ON chat_leads;
   DROP FUNCTION notify_telegram_on_insert();

2. Удалить edge-функцию notify-telegram (код + деплой).
3. Удалить секрет ADMIN_TELEGRAM_CHAT_ID.
4. Отключить коннектор Telegram от проекта.
```

После применения новые заявки не будут вызывать никаких внешних HTTP-запросов — всё чисто.
