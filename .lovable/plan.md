

## План: прямая отправка в Telegram через токен бота

Перейдём с коннектора на прямой вызов Telegram Bot API с использованием токена, который вы предоставили. Это решит текущую проблему с `chat not found`, потому что мы будем точно знать, какой бот шлёт сообщения.

## Что будет сделано

**1. Серверные секреты (backend)**
- Создать новый секрет `TELEGRAM_BOT_TOKEN` со значением `8232353186:AAH_TMIquHRxu7u4nMkYToF5oen2yBmlrLE`
- Обновить секрет `ADMIN_TELEGRAM_CHAT_ID` на `-5195679235` (если ещё не такое)
- Старый `TELEGRAM_API_KEY` (от коннектора) больше не используем — оставим как есть, не мешает

**2. Edge-функция `notify-telegram`**
- Переписать на прямой вызов `https://api.telegram.org/bot<TOKEN>/sendMessage`
- Добавить поддержку нового источника `"website_form"` для прямой отправки формы (поля: `name`, `phone`, `message`)
- Сохранить обратную совместимость с существующими источниками (`booking`, `contact_message`, `chat_lead`) — они продолжат работать через DB-триггер
- Серверная валидация полей (длина, обязательность) через zod
- Возвращать `{ success: true }` или `{ success: false, error }`
- Формат сообщения для нового источника — ровно как вы просили:
  ```
  Новая заявка с сайта нутрициолога
  Имя: ...
  Телефон: ...
  Сообщение: ...
  Дата: ...
  ```

**3. Тест после деплоя**
- Сразу вызову edge-функцию тестовым payload, чтобы убедиться, что сообщение долетает в группу `-5195679235`
- Если `chat not found` повторится — значит бот не добавлен в группу или `chat_id` действительно другой; в этом случае дам инструкцию как получить точный id через `getUpdates`

**4. Что НЕ меняем**
- Дизайн сайта — без изменений
- Существующие формы (`Booking`, `Contacts`, `ChatWidget`) уже отправляют через Supabase → DB триггер → `notify-telegram`. После исправления функции они автоматически заработают, ничего во фронтенде менять не нужно
- Токен бота никогда не попадает в браузер — только в серверной edge-функции через `Deno.env.get("TELEGRAM_BOT_TOKEN")`

```text
Текущая (сломанная) схема:
форма → БД → DB trigger → notify-telegram → connector gateway → бот коннектора (не тот)

Новая схема:
форма → БД → DB trigger → notify-telegram → api.telegram.org/bot<TOKEN>/sendMessage → ваш бот
```

- Edge-функция: `supabase/functions/notify-telegram/index.ts` (полная замена тела)
- Новый секрет: `TELEGRAM_BOT_TOKEN`
- CORS уже настроен корректно
- Существующая `db function notify_telegram_on_insert` остаётся без изменений — она просто вызывает ту же edge-функцию
